# v1.0 完工标准

本文是本项目 v1.0 的权威完成标准。以后判断项目是否完成，不以聊天记录、主观感觉或“功能看起来很多”为准，而以本文的要求和证据为准。

v1.0 的目标是：做成一个本地运行的美股历史盲测训练系统，帮助新手训练证据、仓位、计划、情绪、风控、复盘和长期改进。它不是实时交易系统，也不是券商级模拟盘。

## 范围边界

v1.0 必须包含：

- 本地网页训练器。
- 历史日线回放。
- 强盲测训练和随机考试。
- 买入、卖出、观望、限价单和挂单取消。
- 新手风控、资金结算、公司行动和复盘。
- 至少 20 个开箱可练训练案例。
- 案例包导入导出、完整备份和恢复。
- 长期画像、错题补练、课程路径和训练队列。
- 自动化自检和真实浏览器回归。

v1.0 不包含：

- 实时行情持续同步。
- 真实券商下单。
- 分钟级盘口、期权、融资、做空和高频策略。
- 自动抓全网新闻。
- 云端多用户账户。
- 复杂数据库、登录系统或微服务架构。

这些可以放到 v2 或更后面，不允许用它们拖延 v1.0。

## 完工要求

### V1-01 训练体验完整

必须做到：

- 用户打开 `index.html` 后可以直接开始训练。
- 可以选择案例、开始随机盲测、逐日推进、播放、加速、暂停、跳到关键事件。
- 可以记录买入、卖出和观望。
- 可以提交市价单和限价单。
- 限价单可以等待、成交、过期和主动取消。
- 训练中可以查看价格图、持仓、历史记录、新闻、事件、任务和风险仪表盘。
- 训练结束可以揭晓案例并生成复盘。

验收证据：

- `npm run browser:smoke` 必须通过。
- 浏览器回归必须覆盖真实买入、卖出、观望、限价单成交、限价单过期和限价单取消。
- 移动端横向溢出必须为 `0px`。

### V1-02 盲测严肃性完整

必须做到：

- 训练前隐藏真实公司、真实代码、真实日期和真实事件名。
- 考试模式隐藏案例主题、标的名称、课程讲义和关键节点原因。
- 随机盲测自动进入考试模式。
- 随机盲测尽量避开当前案例和最近练过的案例。
- 复盘时单独审计盲测是否被污染。
- 长期画像必须区分有效随机盲测、普通练习和失真盲测。
- 案例包导入时必须检查文件名和内容是否剧透。

验收证据：

- `tests/smoke.js` 必须覆盖考试模式遮罩、随机盲测、盲测完整性、失真盲测和案例包防剧透体检。
- `tools/validate-case-package.mjs --strict` 必须能拦截明显泄露真实代码、真实年份、真实来源或结果暗示的案例包。
- `docs/BLIND_TRAINING_POLICY.md` 必须存在并说明防剧透规则。

### V1-03 案例库达到课程级

必须做到：

- 开箱内置案例不少于 20 个。
- 内置案例必须覆盖：ETF 系统性风险、快速反弹、慢熊、单股事件、财报预期差、财报超预期、宏观利率、CPI/Fed 数据日、银行或流动性危机、行业轮动、热门股追高、防御配置、心理纪律、公司行动、订单执行和 T+1 结算。
- 每个案例必须至少包含：行情、训练新闻或事件、课程讲义、任务目标、复盘来源或训练说明。
- 案例不能只是价格序列，必须有明确要训练的能力。
- starter 经典案例库可以作为匿名训练形态，但必须能被构建、校验、打包和内置同步。

验收证据：

- 首屏浏览器回归必须显示至少 20 个案例。
- 当前 v1.0 目标案例数为 23 个：9 个真实历史行情案例、3 个合成心理训练案例、11 个内置匿名经典形态案例。
- `tools/audit-case-coverage.mjs --manifest cases/starter-classic-library.json --strict` 必须通过。
- `tools/sync-starter-bundle.mjs --check` 必须通过。
- `tests/smoke.js` 必须验证每个内置案例都有训练任务和课程模块。

### V1-04 新手风控必须硬约束

必须做到：

- 有单股仓位上限、行业仓位上限、现金底线、每日交易次数上限。
- 禁止融资、做空、期权和杠杆 ETF。
- 买入前必须写理由、信息、情绪、信心、最大可接受亏损、失效条件和盈利计划。
- 每笔决策必须记录证据来源。
- 账户触发风险冷却后，必须限制新增买入。
- 流动性、隔夜跳空、事件日风险、未结算资金和现金账户纪律必须进入下单教练。

验收证据：

- `tests/smoke.js` 必须覆盖下单校验、风险冷却、流动性拦截、资金结算提醒、事件日风险、隔夜跳空、证据来源和盈利计划。
- `core/orders.js`、`core/portfolio.js`、`core/runtime.js` 必须有独立 smoke。

### V1-05 资金、结算和公司行动可信

必须做到：

- 美元交易、港币入金口径和换汇成本必须显示。
- 卖出后资金进入 T+1 未结算卖出款。
- 到期结算后现金可用。
- 买入动用未结算资金必须记录。
- 提前卖出未付款 lot 必须标记 good faith 类风险。
- 现金分红必须进入现金。
- 股息预扣税必须显示毛分红、预扣税和净到账。
- 拆股或复权口径事件必须提醒不要重复调整。

验收证据：

- `tests/runtime-smoke.js` 必须覆盖结算、分红、拆股和挂单推进。
- `tests/smoke.js` 必须覆盖资金结算、现金账户纪律、公司行动和股息税费复盘。

### V1-06 复盘系统必须能指导下一轮

必须做到：

- 揭晓后生成完整复盘，不只显示赚亏。
- 复盘必须包含：资料包、事件证据时间线、新闻校准、总评分、盲测完整性、风险冷却、决策质量、资金与汇率、资金结算、订单执行、隔夜跳空、事件日风险、流动性、公司行动、持仓触发、信心校准、失效条件、盈利计划、错误根因、交易账本、盈亏归因、逐笔审计、反思、补练处方、基准对比和高级诊断。
- 复盘后必须能保存反思。
- 复盘后必须能启动补练。
- 复盘必须能导出 Markdown。

验收证据：

- `tests/ui-review-summary-smoke.js` 必须覆盖复盘摘要主要区块。
- `npm run browser:smoke` 必须覆盖揭晓复盘和 Markdown 下载。
- `core/review.js` 必须有独立 smoke。

### V1-07 长期画像和课程路径完整

必须做到：

- 长期画像必须记录每轮训练结果。
- 必须显示纪律、任务通过率、常见错误、情绪倾向、案例掌握度、样本质量、真实考试成绩、市场场景覆盖、实盘前准备度、训练趋势和能力雷达。
- 必须有错题本。
- 必须能从错题卡启动补练。
- 必须有间隔复训日程。
- 必须有今日训练计划和弱项自动排课。
- 课程路径必须有 Level 1/2/3 解锁规则。
- Level 2 必须要求 Level 1 合格，Level 3 必须要求 Level 1 全部合格并完成至少 3 个 Level 2。
- 扩展案例可以自由补练，但不能阻塞主课程路径。

验收证据：

- `tests/course-smoke.js` 必须覆盖课程路径、解锁、掌握度和随机盲测选题。
- `tests/ui-profile-insights-smoke.js` 必须覆盖长期画像主要卡片。
- `tests/ui-training-plan-smoke.js` 必须覆盖今日训练计划。

### V1-08 数据导入导出可靠

必须做到：

- 支持单标的 CSV。
- 支持多标的 CSV。
- 支持完整 JSON 案例包。
- 支持案例库 bundle 一次导入多个案例。
- 支持直接选择本地 `.json` 文件，且可以多选。
- 支持当前案例包导出。
- 支持自定义案例库导出。
- 支持完整备份导出和恢复。
- 重复导入必须按内容指纹跳过。
- 案例内容不能和个人训练记录混在一起导出。

验收证据：

- `tests/importers-smoke.js` 必须覆盖导入解析和质量体检核心逻辑。
- `tests/smoke.js` 必须覆盖 CSV、多标的、JSON 案例包、bundle、重复导入、备份恢复。
- `npm run browser:smoke` 必须覆盖文件选择导入、案例包导出、案例库导出和备份恢复。

### V1-09 工程质量可维护

必须做到：

- 不能把所有逻辑继续塞进一个不可维护的主文件。
- 核心业务逻辑必须逐步模块化。
- 至少以下模块必须存在并有测试或被总自检覆盖：
  - `core/case-library-coverage.js`
  - `core/trade-ledger.js`
  - `core/orders.js`
  - `core/portfolio.js`
  - `core/course.js`
  - `core/runtime.js`
  - `core/review.js`
  - `core/importers.js`
  - `state/session.js`
  - `ui/training-plan.js`
  - `ui/profile-insights.js`
  - `ui/review-summary.js`
  - `ui/events.js`
- starter bundle 内置数据必须可同步，不能手工漂移。
- 代码不得有明显语法错误、调试输出和文档漂移。

验收证据：

- `npm test` 必须通过。
- `tools/check-all.mjs` 必须包含语法检查、核心 smoke、UI smoke、工具链 smoke、源码卫生和文档卫生。
- `tools/sync-starter-bundle.mjs --check` 必须通过。
- `tools/audit-v1-readiness.mjs` 必须通过。
- `tools/audit-v1-evidence.mjs` 必须通过，并逐条覆盖 V1-01 到 V1-10 的证据矩阵。

### V1-10 文档足够让新手自己使用

必须做到：

- `README.md` 必须说明运行方式、核心功能、导入导出、案例包流程和测试命令。
- `PROJECT_STATUS.md` 必须说明当前状态、限制、风险边界和下一步。
- `docs/CONTENT_PIPELINE.md` 必须说明案例生产流程。
- `docs/BLIND_TRAINING_POLICY.md` 必须说明盲测防剧透规则。
- 本文件必须说明 v1.0 完成标准。

验收证据：

- `tools/check-all.mjs` 的 docs hygiene 必须通过。
- `tools/audit-v1-readiness.mjs` 必须检查上述文档存在并被 README 或状态文档引用。
- `tools/audit-v1-evidence.mjs` 必须检查每个 v1.0 条目都有对应的测试、浏览器回归、模块或文档证据。

## 必须通过的命令

判断 v1.0 完成前，至少必须运行并通过：

```bash
npm test
npm run browser:smoke
node tools/audit-v1-evidence.mjs
```

其中：

- `npm test` 证明语法、核心逻辑、UI 模块、工具链、案例库、源码卫生和文档卫生没有明显回归。
- `npm run browser:smoke` 证明真实浏览器里关键训练路径可以运行。
- `node tools/audit-v1-evidence.mjs` 证明 V1-01 到 V1-10 都有明确证据，不靠主观印象判断完成。

## 完成判定

v1.0 只有在以下条件同时满足时才算完成：

1. 本文件 V1-01 到 V1-10 的要求都有明确证据。
2. `npm test` 通过。
3. `npm run browser:smoke` 通过。
4. 当前已知限制都属于“v1.0 不包含”的范围，而不是 v1.0 必须功能缺失。
5. 没有为了通过测试而降低本文标准。
6. 没有只靠人工印象或单次窄测试声称完成。

如果某条证据不完整，就不能标记 v1.0 完成，只能继续补功能、补测试或补文档。
